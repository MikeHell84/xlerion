# Desarrollo Web y Móvil / Web & Mobile Development

## ES — Resumen (máx 2 páginas)

**Alcance**: sitios corporativos, e‑commerce, landing pages, apps híbridas.  
**Fases**: Discovery → Diseño → Desarrollo → Integración → QA → Deploy → Post‑lanzamiento.  
**Duración típica**: 2–20 semanas según subservicio y tamaño.  

**Supuestos**

- Accesos y contenidos disponibles antes de S1.
- Definición de objetivos de conversión para landings.
- APIs documentadas para integraciones.

**Riesgos**

- Requisitos incompletos → Sprint 0 obligatorio.
- Cambios de alcance tardíos (buffer 15–30%).
- Integraciones de pago/CRM con tiempos externos.

**Recomendaciones**

- Recalibrar cada 3–4 sprints en proyectos >10 semanas.
- Incluir pruebas de rendimiento (LCP < 2.5s) en landings.
- Para apps: pruebas multiplataforma y plan de hotfix.

## EN — Summary (max 2 pages)

**Scope**: corporate sites, e‑commerce, landing pages, hybrid apps.  
**Phases**: Discovery → Design → Development → Integration → QA → Deploy → Post‑launch.  
**Typical duration**: 2–20 weeks depending on subservice and size.  

**Assumptions**

- Content and access available before S1.
- Conversion goals defined for landings.
- APIs documented for integrations.

**Risks**

- Incomplete requirements → mandatory Sprint 0.
- Late scope changes (15–30% buffer).
- External timelines for payment/CRM integrations.

**Recommendations**

- Recalibrate every 3–4 sprints for projects >10 weeks.
- Include performance checks (LCP < 2.5s) for landings.
- For apps: cross‑platform testing and hotfix plan.

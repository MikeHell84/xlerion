# Software Empresarial / Enterprise Software

## ES — Resumen (máx 2 páginas)

**Alcance**: ERP, CRM, POS, automatización de procesos, integraciones de pago.  
**Fases**: Discovery → Diseño → Desarrollo → Integración → QA → Deploy.  
**Duración típica**: 3–36 semanas según módulos, integraciones y migración.  

**Supuestos**

- Procesos de negocio documentados.
- Datos legacy accesibles y auditables.
- Aprobaciones de usuarios clave en UAT.

**Riesgos**

- Migraciones legacy con datos inconsistentes.
- Cambios de proceso en medio del desarrollo.
- Dependencias con proveedores externos.

**Recomendaciones**

- Sprint 0 obligatorio si procesos no están definidos.
- Incluir piloto de migración antes del Go‑live.
- Checkpoints de recalibración cada 3–4 sprints.

## EN — Summary (max 2 pages)

**Scope**: ERP, CRM, POS, process automation, payment integrations.  
**Phases**: Discovery → Design → Development → Integration → QA → Deploy.  
**Typical duration**: 3–36 weeks based on modules, integrations, migration.  

**Assumptions**

- Business processes documented.
- Legacy data accessible and auditable.
- Key users available for UAT approvals.

**Risks**

- Legacy migrations with inconsistent data.
- Process changes during development.
- External provider dependencies.

**Recommendations**

- Sprint 0 mandatory if processes are undefined.
- Run a migration pilot before Go‑live.
- Recalibration checkpoints every 3–4 sprints.

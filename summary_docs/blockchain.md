# Tecnologías en Blockchain / Blockchain Technologies

## ES — Resumen (máx 2 páginas)

**Alcance**: smart contracts, tokenización, integraciones, arquitectura escalable.  
**Fases**: Discovery → Diseño → Desarrollo → QA → Deploy.  
**Duración típica**: 4–16 semanas según contratos y auditoría.  

**Supuestos**

- Requisitos legales definidos.
- Acceso a testnet y proveedores.
- Criterios de auditoría establecidos.

**Riesgos**

- Retrasos por auditorías externas.
- Costos de gas variables.
- Vulnerabilidades críticas en contratos.

**Recomendaciones**

- Sprint 0 obligatorio si requisitos no están claros.
- Auditoría externa para mainnet.
- Recalibración cada 3–4 sprints en proyectos largos.

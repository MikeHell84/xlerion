# Roadmap — Desarrollo de Videojuegos

Service: Desarrollo de Videojuegos
Sprint duration: 3 semanas (recomendado para producción)
Project types: Prototipo, Indie, AA/AAA

## Cómo leer este roadmap

- Separar tracks: Programación, Arte, Audio, QA. Cada sprint incluye integraciones entre tracks.

---

## Indie pequeño (ejemplo: 10 sprints de 3 semanas)

Sprint 1–2 — Prototipo jugable

- Objetivos: Definir core loop, prototipo jugable del nivel 1.
- Entregables: Build jugable con input básico, documento técnico.
- Aceptación: Playtest con 5 usuarios; métricas iniciales de retención.

Sprint 3–7 — Producción de contenido

- Objetivos: Desarrollo de niveles, assets, IA básica, integración audio.
- Entregables: Niveles integrados, assets importados, systems funcionales.
- Aceptación: Cada sprint: 80% de historias planeadas completadas; integración sin regressions.

Sprint 8–9 — Pulido y QA

- Objetivos: Ajuste de balance, optimizaciones, bugfixing y tests de rendimiento.
- Entregables: Candidate build para QA, reporte de bugs críticos.
- Aceptación: Bugs críticos ≤ 5, performance objetivo alcanzado.

Sprint 10 — Lanzamiento y Post‑launch

- Objetivos: Preparar store, marketing básico, soporte inicial.
- Entregables: Build final, plan de parches, lista de issues conocidos.
- Aceptación: Publicación en target store (según checklist), plan de soporte activado.

---

## Variables de esfuerzo

- Cantidad de niveles: cada 3–4 niveles → +1 sprint.
- Número de personajes con animaciones: +0.5 sprint por cada 3 personajes adicionales.
- Plataformas objetivo: +1 sprint por plataforma adicional (ej. consola).
- Localización: +0.5 sprint por idioma.

## Notas

- Mantener integración continua (build diario) y pruebas automáticas donde sea posible.
- Alinear sprints cross‑discipline: sincronizar entregas de arte y código cada sprint.
